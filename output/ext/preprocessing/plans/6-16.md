Purpose of this document:

This document defines the engineering specification for preprocessing the Botswana Imports (6-16) dataset before it enters merge.py and the downstream modelling workflow.

Unlike the analysis report, which only summarizes the imported spreadsheet, this document specifies every preprocessing operation that preprocessing.py shall implement. The objective is to transform the imports statistical publication into a normalized monthly analytical dataset suitable for integration with every other economic dataset within the MmarakaAI pipeline.

This preprocessing stage prepares the dataset exclusively for merging. Feature engineering, lag generation, rolling statistics, scaling, encoding, dimensionality reduction and machine learning transformations are intentionally excluded.

# Dataset Overview

The Botswana Imports dataset contains official import statistics organized according to commodity classifications.

The workbook typically includes

• report titles

• reporting periods

• explanatory notes

• commodity classifications

• merged cells

• blank separator rows

• repeated section headings

• monthly import values

Although these structures improve readability, they prevent direct analytical processing.

The preprocessing objective is therefore to reconstruct the publication into a relational time-series dataset compatible with all other monthly datasets used within MmarakaAI.

This dataset should never be treated as machine-ready until structural transformation has been completed.

# Structural Changes

## 1. Publication Reconstruction

The workbook shall first be interpreted as a statistical publication rather than an analytical dataset.

preprocessing.py shall remove

• publication titles

• report metadata

• explanatory paragraphs

• notes

• source references

• footnotes

• blank rows

• blank columns

• repeated section headers

• merged formatting cells

These elements contain no analytical information and must be removed before any cleaning begins.

---

## 2. Header Detection

The imported dataframe may contain several unnamed columns generated from merged spreadsheet cells.

The preprocessing pipeline shall automatically identify the first row containing the actual commodity names and monthly observations.

That row shall become the dataframe header.

Rows preceding the detected header shall be discarded.

Remaining unnamed columns shall be removed only after confirming they contain no analytical values.

---

This normalized structure is required throughout the MmarakaAI pipeline.

---

## 4. Cleaning

Cleaning begins only after structural reconstruction.

Cleaning operations include

• remove duplicated observations

• remove repeated publication headers

• trim whitespace

• standardize commodity names

• validate numerical import values

• remove empty analytical rows

Publication formatting artefacts shall never be interpreted as missing economic observations.

---

## 5. Datetime Normalization

Monthly reporting periods shall be converted into pandas datetime format.

The first day of each month shall represent the monthly observation.

This creates consistency across all monthly datasets.

---

## 7. Missing Value Assessment

Missing values detected before structural reconstruction shall not immediately be interpreted as missing analytical observations.

Many apparent missing cells originate from

• merged spreadsheet formatting

• publication separators

• blank formatting columns

• section breaks

Only missing values remaining after reconstruction shall be evaluated.

Automatic imputation shall not be performed during preprocessing.
